# 23-ПО-2 Блошкин Алексей ЛР1
## Торговый автомат 

## Описание
Лабораторная работа №1 по курсу Функционального программирования.
Реализован сценарий "Торговый автомат" (Вариант 1) с использованием базовых монад: `Reader`, `Writer`, `State`, `IO`.

### Роли монад в проекте
- **Reader**: используется для прокидывания конфигурации автомата (допустимые монеты, цены, правила скидок) без использования глобальных переменных.
- **Writer**: логирует внесение монет и попытки покупки товара, формируя аккуратный список сообщений о произведённых действиях.
- **State**: инкапсулирует и хранит текущее состояние мира (внесенные монеты, остатки товаров в автомате, накопленную выручку, кассу автомата).
- **IO**: обеспечивает безопасную и декларативную работу с консолью, организуя изолированный от логики цикл взаимодействия с пользователем.

## Архитектурные особенности 
- **Безопасное меню на ADT**: Обработка действий пользователя построена на базе строго типизированного алгебраического типа данных `MenuCommand`. Программа полностью избавлена от сырых проверок строк в стиле `input match { case "1" => ... }`. Парсинг текстовых команд строго изолирован от их последующей интерпретации.
- **Чистота эффектов**: Взаимодействие с консолью (ввод-вывод) полностью инкапсулировано внутри ленивого `IO` с явным указанием кодировки UTF-8. Логика предметной области спроектирована исключительно на чистых функциях.

## Структура проекта
1. `src/main/scala/vending/monads/` — определения монад, написанных с нуля (`Monad`, `Reader`, `Writer`, `State`, `IO`).
2. `src/main/scala/vending/domain/` — предметные типы, настройки и функции, реализующие бизнес-логику автомата через построенные монады.
3. `src/main/scala/vending/Main.scala` — основной сценарий взаимодействия и интерпретатор команд меню.

## Запуск
